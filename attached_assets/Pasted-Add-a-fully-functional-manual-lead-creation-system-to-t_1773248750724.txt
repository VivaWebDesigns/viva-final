Add a fully functional manual lead creation system to the Viva Web Designs internal platform in a non-destructive way. Do not remove, replace, or break any existing CRM, Sales Pipeline, Tasks, Client Onboarding, Payments, Notifications, import/export, filtering, search, language toggle, or current data flows. Extend the current system cleanly.

GOAL:
Users must be able to manually add a new lead without importing, and that lead must appear correctly in both the CRM and the Sales Pipeline.

IMPLEMENT THE FOLLOWING:

1) CRM PAGE: ADD LEAD BUTTON
On the CRM Leads page, add a clear primary button in the top-right action area near Export and Import called:
+ Add Lead

2) OPEN A CREATE LEAD MODAL OR DRAWER
When the user clicks + Add Lead, open a polished modal or slide-over form that matches the current platform UI.

Title:
Create New Lead

Fields:
- First Name (required)
- Last Name (required)
- Business Name (optional)
- Business Trade (required)
- Phone (required)
- Email (optional)
- Website (optional)
- Source (required dropdown with only these options: Website, Outreach)
- Notes (optional multiline)

Buttons:
- Cancel
- Save Lead

3) VALIDATION
Before saving:
- Require First Name
- Require Last Name
- Require Business Trade
- Require Contact Number
- Require Source
Show inline validation errors under the missing fields.
Do not allow the form to submit until required fields are complete.

4) DATA MODEL
Create or extend a reusable lead record model so all lead-related pages use the same source of truth.
Each lead should include at minimum:

- id
- firstName
- lastName
- businessName
- businessTrade
- phone
- email
- website
- source
- notes
- status
- createdAt
- updatedAt

If the system already has a lead model, extend and reuse it instead of creating a conflicting second model.

5) DEFAULT STATUS
When a lead is manually created, automatically assign:
status = "New Lead"

This status must integrate cleanly with the Sales Pipeline so the lead starts in the first pipeline stage.

6) SAVE BEHAVIOR
When Save Lead is clicked and validation passes:

- create the lead record
- persist it using the same existing storage/state architecture the platform already uses
- immediately add it to the CRM list without refresh
- immediately make it available in the Sales Pipeline under New Lead without refresh
- close the modal
- show a success toast:

Lead created successfully

7) CRM INTEGRATION
The newly created lead must fully work inside the CRM page:

- searchable by name, business name, business trade, phone, email, and website if applicable
- filterable by status
- filterable by source
- included in any existing list rendering, counters, empty-state replacement, and export logic

If the page currently shows “0 leads” or “No leads found,” that should update automatically once a lead is added.

8) SALES PIPELINE INTEGRATION
The same lead must also appear in the Sales Pipeline automatically under:

New Lead

The pipeline card/item should display useful summary information such as:

- Business Name (if available)
- Business Trade
- Contact Full Name
- Phone
- Email
- Source

Do not create duplicate records between CRM and Sales Pipeline. They should reference the same underlying lead/deal entity or a clean linked structure, depending on the current app architecture. Avoid creating separate disconnected objects that can drift out of sync.
