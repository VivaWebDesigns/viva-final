Fix the EN/ES language system in the demo builder and generated preview sites. The English side is working well enough, but the Spanish side is still badly broken and much of the client-facing site remains in English when ES is selected. This needs a careful, comprehensive, non-destructive fix focused on making Spanish mode truly complete and reliable across the entire generated website.

IMPORTANT RULES
- Be non-destructive.
- Do not break any working English behavior.
- Do not break preview generation, routing, styling, forms, demo builder flow, package logic, image logic, or state management.
- Do not redesign the UI.
- Do not remove any working features.
- Reuse the current architecture where possible.
- If a refactor is required, keep it targeted and safe.
- Do not hardcode quick one-off Spanish strings in scattered components.
- Fix the root cause, not just a few visible pages.
- Preserve all existing dynamic company/trade/service behavior.

GOAL
When the language toggle is set to ES, the entire generated website must be in proper Spanish with no English leakage anywhere client-facing. English mode must continue to work exactly as it does now.

PRIMARY PROBLEM TO SOLVE
Right now the language toggle appears to switch only some text, while much of the site still renders English. This strongly suggests that:
- not all client-facing strings are connected to the translation system
- some components still contain hardcoded English
- some generated content is being built from English-only defaults
- some nested/shared components are bypassing the current language state
- some service/company-specific text may be generated only in English
- some form labels/options/placeholders may still use old hardcoded arrays
- some fallback text may still default to English even in ES mode
