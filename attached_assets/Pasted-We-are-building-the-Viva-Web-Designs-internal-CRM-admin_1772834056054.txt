We are building the Viva Web Designs internal CRM/admin platform in Replit.

PROJECT CONTEXT
- This is an internal business platform for Viva Web Designs.
- It is NOT multi-tenant.
- Stack: full-stack React + TypeScript frontend, Express + TypeScript backend, Node.js, PostgreSQL, Drizzle ORM.
- Authentication uses BetterAuth.
- Frontend UI stack must use Tailwind CSS, shadcn/ui, Framer Motion, and Lucide React.
- Cloudflare R2 is the exclusive storage provider for all file/object storage. Nothing should be permanently stored locally.
- Integrations area must support Stripe, Mailgun, OpenAI, and Cloudflare R2.
- User roles:
  - Admin
  - Developer
  - Sales Rep
- All work in this prompt must be NON-DESTRUCTIVE. Do not remove or break existing functionality. Extend safely.
- Maintain modular architecture, organized file structure, and performance optimization.
- Break up route files, service files, and frontend feature files by domain/function. Avoid monolithic “god files.”
- Every change must include documentation updates in the internal App Docs library.

PRIMARY GOAL
Build the foundational architecture for the internal platform so future feature prompts have a stable base.
