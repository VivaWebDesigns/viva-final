Fix 1 — Query-backed detail view
Problem:
The detail panel/drawer is currently driven by the local
row snapshot from the table. After Skip or Convert, the
detail view can become stale until the user reopens it.

Required fix:
- track selected record by ID, not by full row object
- fetch the selected record from:
  GET /api/marketplace/pending-outreach/:id
- use a dedicated TanStack Query keyed by selectedId
- render detail from that query result, not from the
  original row snapshot
- after Skip or Convert, invalidate:
  - list query
  - summary query
  - selected detail query (if selectedId exists)

Recommended query key pattern:
- ["/api/marketplace/pending-outreach", filters...]
- ["/api/marketplace/pending-outreach/summary"]
- ["/api/marketplace/pending-outreach", selectedId]

The selected detail view must always reflect the current
server truth after actions.
