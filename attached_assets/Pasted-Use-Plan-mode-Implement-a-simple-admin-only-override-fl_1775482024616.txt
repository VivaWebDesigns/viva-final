Use Plan mode.

Implement a simple, admin-only override flow for Marketplace
precheck failures caused by Hispanic name scoring, plus admin-only
actions to permanently add missing names to our saved Hispanic name
lists.

Scope and constraints:
- Keep this simple.
- Do NOT add audit tables.
- Do NOT add timestamps/history tracking UI.
- Do NOT add complex admin dashboards.
- Do NOT weaken the default protection for non-admin users.
- Do NOT allow overriding duplicate detection or missing required

- Only allow override for the specific reason:
  name_score_below_threshold

Goal:
When the extension precheck says a seller is "not Hispanic enough"
but I know they clearly are, I want admin-only actions that let me:
1. override this one record and continue immediately
2. add the parsed first name and/or surname to the existing saved
   Hispanic name lists used by the backend scoring logic

High-level behavior:
- Non-admin users:
  - no change to current behavior
  - they only see the normal skip/fail behavior
- Admin users:
  - if precheck fails ONLY because of
    reason = name_score_below_threshold
  - show extra actions in the extension popup:
    - Override
    - Add first name
    - Add surname
    - Add both
- Override:
  - bypasses only the name-score failure for this one attempt
  - does NOT bypass duplicate checks
  - does NOT bypass missing seller name
  - does NOT bypass missing seller profile URL
  - does NOT bypass missing profile URL / required fields
- Add first/surname/both:
  - updates the real saved Hispanic name lists already used in the
    backend scoring code
  - normalize input before saving
  - avoid duplicates
  - after saving, either rerun precheck automatically or continue
    with a clean success path if that is simpler and safer

Important architecture:
- Do NOT let the Chrome extension directly edit local files
- The extension must call backend admin-only API route(s)
- Backend performs:
  - auth/role check
  - normalization
  - persistence/update of the existing name lists
  - safe dedupe
- Keep the existing list source of truth intact and extend it
  cleanly
